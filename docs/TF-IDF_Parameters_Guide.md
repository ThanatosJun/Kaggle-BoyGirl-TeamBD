# TF-IDF 參數調整指南

## 📊 數據集特性

**你的數據情況**：
- **樣本數量**: ~400+ 筆資料
- **文本長度**: 10-15 個單詞（短文本）
- **特徵欄位**: `self_intro`（自我介紹）

---

## ⚙️ 參數優化分析

### 1️⃣ `ngram_range=(1, 1)` ✅ 推薦

**原設定**: `(1, 2)` - unigram + bigram  
**優化後**: `(1, 1)` - 只用 unigram

**為什麼修改？**

| 指標 | Unigram | Unigram + Bigram |
|------|---------|------------------|
| **短文本（10-15詞）** | ✅ 充足 | ❌ 過於稀疏 |
| **特徵空間** | ~50-100 維 | ~200-500 維（爆炸增長）|
| **數據需求** | 400+ 樣本 OK | 需要 1000+ 樣本 |
| **過擬合風險** | 低 | **高** ⚠️ |

**計算範例**：
- 若有 100 個獨特單詞
- Unigram: 100 個特徵
- Bigram: 100 × 99 / 2 = **4,950 個潛在特徵**（大多只出現1次）

**建議**：
- ✅ 保持 `(1, 1)` 給小數據集
- 🔄 若數據量 > 1000，可嘗試 `(1, 2)`

---

### 2️⃣ `min_df=1` ✅ 推薦

**原設定**: `min_df=2` - 至少出現在 2 個文檔  
**優化後**: `min_df=1` - 保留所有出現的詞

**為什麼修改？**

| min_df | 400 樣本場景 | 影響 |
|--------|-------------|------|
| `min_df=1` | 保留所有詞 | ✅ 最大化信息保留 |
| `min_df=2` | 過濾 ~25-30% 的詞 | ⚠️ 損失罕見但關鍵的特徵 |
| `min_df=5` | 過濾 ~50% 的詞 | ❌ 過度過濾 |

**實際案例**：
```
"我喜歡打籃球" → 若 "籃球" 只在 1 個文檔出現
min_df=2 → 丟棄 "籃球"（可能是性別判斷的關鍵詞！）
min_df=1 → 保留 "籃球"
```

**建議**：
- ✅ 小數據集（< 1000）用 `min_df=1`
- 🔄 大數據集（> 5000）用 `min_df=2-5`

---

### 3️⃣ `max_df=0.95` ✅ 推薦

**原設定**: `max_df=0.8` - 過濾出現在 > 80% 文檔的詞  
**優化後**: `max_df=0.95` - 過濾出現在 > 95% 文檔的詞

**為什麼修改？**

| max_df | 400 樣本 = 過濾閾值 | 被過濾的詞 |
|--------|-------------------|-----------|
| `max_df=0.8` | > 320 個文檔 | "的"、"是"、"我"、**"喜歡"** 😱 |
| `max_df=0.95` | > 380 個文檔 | "的"、"是" ✅（僅極端停用詞）|

**問題**：0.8 太嚴格，可能過濾掉有用的高頻詞

**實際案例**：
```
假設 80% 的人都寫 "我喜歡..."
max_df=0.8 → 丟棄 "喜歡"（但 "喜歡" 後接的對象可能有性別差異！）
max_df=0.95 → 保留 "喜歡"
```

**建議**：
- ✅ 短文本用 `max_df=0.95`（保留更多資訊）
- 🔄 長文本（> 100詞）用 `max_df=0.85`

---

### 4️⃣ `max_features=50` ✅ 推薦

**原設定**: `max_features=100`  
**優化後**: `max_features=50`（可根據實驗調整至 50-80）

**為什麼修改？**

| 文本長度 | 建議 max_features | 理由 |
|---------|------------------|------|
| 10-15 詞 | **50-80** | ✅ 最優（避免過擬合）|
| 30-50 詞 | 100-150 | 中等 |
| 100+ 詞 | 200-500 | 長文本 |

**計算邏輯**：
```
短文本詞彙量 ≈ 平均詞數 × 1.5 = 15 × 1.5 = 22.5 個獨特詞/文檔
全數據集獨特詞 ≈ 22.5 × √400 = 450 個獨特詞（估算）
max_features = 獨特詞 × 0.15 ≈ 50-70
```

**特徵數 vs 過擬合風險**：

| max_features | 400 樣本/特徵比 | 風險 |
|-------------|----------------|------|
| 50 | 8:1 | ✅ 安全 |
| 80 | 5:1 | ✅ 可接受 |
| 100 | 4:1 | ⚠️ 邊緣 |
| 150 | 2.7:1 | ❌ 高風險 |

**建議**：
- ✅ 從 50 開始實驗
- 🔄 若模型欠擬合，逐步增加至 80

---

### 5️⃣ `sublinear_tf=True` ✅ 保持

**作用**: 使用 `1 + log(tf)` 替代原始詞頻

**為什麼保留？**

短文本特性：
- 某詞出現 1 次 vs 2 次，差異巨大（文本總長度才 10 詞！）
- 需要平滑處理，避免詞頻主導

**效果對比**：
```
假設某詞在短文本中出現 3 次：

sublinear_tf=False → tf = 3
sublinear_tf=True  → tf = 1 + log(3) ≈ 2.1

比例變化：從 3:1 變為 2.1:1（更平滑）
```

---

### 6️⃣ `token_pattern=r'(?u)\b\w+\b'` ✅ 新增

**作用**: 支援中英文混合，保留單字符詞（如 "I", "我"）

**原因**：
- 預設 pattern 會過濾單字符
- 中文/英文短文本中，單字符詞可能有意義（如 "I love..."）

---

## 🎯 最終推薦配置

### 適用場景：短文本（10-15詞）+ 小數據（400+樣本）

```python
TfidfVectorizer(
    max_features=50,           # 短文本建議 50-80
    ngram_range=(1, 1),        # 只用 unigram（避免稀疏）
    min_df=1,                  # 保留所有詞（小數據集）
    max_df=0.95,               # 只過濾極端高頻詞
    sublinear_tf=True,         # 平滑詞頻（短文本必須）
    token_pattern=r'(?u)\b\w+\b'  # 中文友好
)
```

**配置檔設定**：
```yaml
features:
  text_cols: ["self_intro"]
  text_embedding_method: "tfidf"
  tfidf_max_features: 50  # 可調整: 50, 60, 70, 80
```

## 📈 參數調整實驗計劃

### 建議實驗序列

| 實驗 | max_features | ngram_range | min_df | max_df | 目標 |
|------|-------------|-------------|--------|--------|------|
| Exp 3-2a | 50 | (1, 1) | 1 | 0.95 | 基準線 |
| Exp 3-2b | 70 | (1, 1) | 1 | 0.95 | 測試更多特徵 |
| Exp 3-2c | 50 | (1, 2) | 1 | 0.95 | 測試 bigram |
| Exp 3-2d | 50 | (1, 1) | 2 | 0.90 | 測試更嚴格過濾 |

### 如何判斷最優？

1. **F1-Score 提升** → 選擇最高的
2. **過擬合檢查**: 
   - CV Std < 0.03 ✅ 穩定
   - CV Std > 0.05 ❌ 過擬合
3. **特徵重要性**: 查看 TF-IDF 詞彙是否合理（人工檢查）


```python
from sklearn.feature_extraction.text import ENGLISH_STOP_WORDS
chinese_stop_words = {'的', '了', '在', '是', '我', '有', '和'}
stop_words = set(ENGLISH_STOP_WORDS) | chinese_stop_words

TfidfVectorizer(stop_words=stop_words, ...)
```

```python
# 若 max_features=80 仍過擬合
```

## ✅ 總結

**你的原始設定問題**：
1. ❌ `min_df=2` 過濾太多（小數據集損失大）
2. ❌ `max_df=0.8` 過濾過嚴（丟失有用高頻詞）
3. ❌ `ngram_range=(1,2)` 特徵空間爆炸（短文本不需要）
4. ❌ `max_features=100` 對短文本略多（риск過擬合）

**優化後設定優勢**：
1. ✅ 保留更多信息（min_df=1）
2. ✅ 避免過度過濾（max_df=0.95）
3. ✅ 降低稀疏度（ngram_range=(1,1)）
4. ✅ 防止過擬合（max_features=50）

**預期效果**：
- TF-IDF 提升 **+0.3-0.8% F1**（相較無文本特徵）
- 若 self_intro 含性別相關信息，效果會更顯著
- 穩定性提升（CV Std 降低）
