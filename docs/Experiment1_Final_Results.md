## 🔍 深度分析

### 為什麼 Median 比 Mean 更優？

**資料集特性**：
- weight 存在 -1000 的極端異常值（明顯輸入錯誤）
- 此類異常值大幅拉低 Mean，但不影響 Median
- 例：`[-1000, 50, 55, 60, 65]` → Mean ≈ 6，Median = 55

**實驗結果驗證**：
```
Median: F1 = 0.9238 ✅
Mean:   F1 = 0.8295 ❌ (下降 10.2%)
```

---

### 為什麼論文中點補值優於資料集 Mean？

**理由**：
1. Mean 易受異常值影響（同上）
2. 論文中點值基於醫學標準（理論基礎強）
3. 論文標準相對保守，避免極端值
4. 與本數據集分布相近

**結果驗證**：
```
論文中點 (方法 2): F1 = 0.9049 ✅
資料集 Mean (方法 1): F1 = 0.8295 ❌ (下降 7.7%)
```

---

### 為什麼分群平均 (star_sign) 效果差？

**問題分析**：
- star_sign（星座）與身高/體重無因果關係
- 分群補值實際上引入了**無關特徵的噪聲**
- 違反假設：不同星座人群的身高/體重沒有顯著差異

**證據**：
- 方法 3 的 CV Std 最高（0.0525），說明穩定性差
- 各 fold 結果差異大，模型過度擬合

**結論**：
```
分群平均 (方法 3): F1 = 0.8108 ❌ (下降 12.1%)
原因：star_sign 無相關性，反而是特徵工程的反大本
```

## 📝 結論與決策

### 最終決策

✅ **採納方法 0（全局 Median）作為實際補值策略**

**四層理由**：

1. **性能最優**
   - F1-Score：0.9238（最高）
   - Accuracy：0.8866（最高）
   - Recall：0.9241（最低誤診率）

2. **穩健性最強**
   - Median 對異常值不敏感
   - CV Std F1 = 0.0274（優秀）
   - 所有 fold 表現一致

3. **實現最簡單**
   - 無須外部信息
   - 訓練集和測試集邏輯一致
   - 無數據洩漏風險

4. **臨床價值最高**
   - Recall 最高（誤診風險最低）
   - 在醫療預測場景中至關重要
